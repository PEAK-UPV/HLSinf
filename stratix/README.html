HLS RELU component as library for OpenCL kernel
HLS component implementation as a library for use in OpenCL kernel

This is the readme file for the OpenCL RELU Library Example.

This document contains information about this design example on how to implement an OpenCL kernel calling a function inside a library that includes the functionality of the kernel.

This example assumes that there is one single Intel FPGA platform on the system. Otherwise the behaviour of the example is undefined.

For more examples, please visit the Intel® FPGA OpenCL™ Design Examples page.

Description

This folder contains the necessary files to generate a library containing a component developed in HLS that is used inside an OpenCL kernel.

The HLS component inside the library performs a RELU operation (RELU kernel).

device/lib sub-folder contains the piece of cpp code that defines a library function 'lib_relu' that performs the RELU operations for data in a floating-point array. In order to be called by the OpenCL kernel, this library function is marked as extern "C" because OpenCL is C-based.

The device kernel file contains a kernel that uses the library function.

The host runs the FPGA kernel and checks result correctness. Results read from FPGA memory are compared with results calculated in the Host.

Two different tools will be used to generate the library from HLS code. FPGA Crossgen and FPGA Libtool

NOTICE

This example involves HLS and OpenCL code, so all environment variables for both developement tools must be set.

Current Hardware System consists of an Intel Stratix 10 MX Developement kit running a BSP v.20.4

Functionality and tools necessary for this example are provided by Quartus Prime Design Software (Quartus II) and Intel® FPGA SDK for OpenCL™. Intel® FPGA SDK for OpenCL™ supports boards running BSP of version 20.4.

NOTICE

Sources of this example include dependencies to "common" and "web" folders source files. These folders are placed in parent directory. Scripts, Makefile and even this README are tuned to search these files on its parent directory. In case you want to move this example folder to another location you must adjust the main Makefile and pass the corresponding variables to compiler and/or linker to find the hls and aocl sources, or you can also copy the necessary folders and keep the structure to your desired destination path.

Initialize Environment

Before performing any other operation, the environment variables for both HLS and OpenCL tools must be initialized with the following command

source ./setenv_iq_20.4.sh

This script sources both OCL and HLS initialization scripts provided alongside with the Quartus Prime Pro Design Software and the Intel® FPGA SDK for OpenCL™. It also sets all the necessary extra environment variables to run the software and compile and link the sources.

NOTICE

This script is ready for Quartus Prime Pro Design Software installed on the following paths

Version Installation Path
20.4 /opt/intelFPGA_pro/20.4

In case that the software is installed in different locations, the script must be updated accordingly.

Compiling the OpenCL Kernel

Before compiling the kernel, and assuming that current directory is

HLSinf/stratix

the library must be built with the following commands:

cd device/src/lib
fpga_crossgen lib_conv2D.cpp --target ocl -o lib_conv2D.aoco
fpga_libtool --target ocl --create lib_conv2D.aoclib lib_conv2D.aoco
cp lib_conv2D.aoclib ..
cd ../../..

Now that the library file is built and resides in device folder, compile the kernel using this library.

The top-level OpenCL kernel file is device/src/k_conv2D.cl.

To compile the OpenCL kernel, run:

aoc -march=emulator -v device/src/k_conv2D.cl -o k_conv2D.aocx -I device/src/lib -L device/src -l lib_conv2D.aoclib [-board=<board>]
cp k_conv2D.aocx bin

Compile the OpenCL kernel may take hours to complete.

where <board> is optional, only necessary for systems with more than one board. In our system, with one single board, it is not necessary.

The -o bin/k_relu.aocx argument is used to place the compiled binary in the location that the host program expects.

If you are unsure of the boards available, use the following command to list available boards:

aoc -list-boards

The board in our system is s10mx_hbm, so the full parameter should be -board=s10mx_hbm.

Compiling for Emulator

To use the emulation flow, the compilation command just needs to be modified slightly:

aoc -march=emulator -v device/src/k_conv2D.cl -o k_conv2D.aocx -I device/src/lib -L device/src -l lib_conv2D.aoclib [-board=<board>]
cp k_conv2D.aocx bin

Compiling for Simulator

To use the simulation flow, the compilation command just needs to be modified slightly:

aoc -march=simulator device/k_relu.cl -o bin/k_relu.aocx -I device/lib -L device -l lib_relu.aoclib [-board=<board>]

Compiling the Host Program

To compile the host program, run:

make

The compiled host program will be located at bin/host.

Running the Host Program

Current BSP only supports emulation mode, thus kernel files must be compiled for emulation

Before running the host program, you should have compiled the OpenCL kernel and the host program. Refer to the above sections if you have not completed those steps.

To run the host program on hardware, execute:

cd bin
./host input.data k_conv2D.aocx

Host Parameters

The host program requires two files:

  1. An input file containg configration for convolutional kernel test.
  2. An OpenCL binary kernel file for stratix board (.aocx).
For this design, both files should be placed in the bin directory.

The general command-line for the host program is:

bin/host <input_file> <kernel_file.aocx>

where the parameters are:

Parameter Type Description
input_file mandatory convolutional kernel configuration parameters.
kernel_file mandatory OpenCL kernel binary file
